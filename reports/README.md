# reports/

Daily and weekly work summaries for the piranav AIOS project.

## Structure

```
reports/
  daily/        One file per working day — YYYY-MM-DD-task-summary.md
  weekly/       (future) Weekly rollup summaries
```

## Rules

- One daily report per day — do not create duplicates
- Check for existing file before creating
- Each report must include: timeline, evidence paths, queryability check, handover notes, pass/fail
- Reports are read-only after the day ends — append a correction note rather than editing history
