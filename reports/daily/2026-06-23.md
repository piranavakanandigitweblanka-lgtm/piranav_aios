### 2. Fix Applied — `sections/banner-gallery.liquid`

**Change:** Added `| strip` before each `| default:` in the banner label assignment chain for both Banner 1 and Banner 2.

**Before (Banner 1):**
```liquid
{% assign banner_1_label = section.settings.main_banner_aria_label | default: section.settings.main_image.alt | default: 'Shop our collection' %}
```

**After (Banner 1):**
```liquid
{% assign banner_1_setting = section.settings.main_banner_aria_label | strip %}
{% assign banner_1_alt = section.settings.main_image.alt | strip %}
{% assign banner_1_label = banner_1_setting | default: banner_1_alt | default: 'Shop our collection' %}
```

Same pattern applied to Banner 2 (`main_banner_aria_label_2` / `main_image_2.alt`).

**Why:** Liquid's `| default:` does not fire on whitespace-only strings (e.g. `"  "`). Without `| strip`, a merchant saving a blank/spaces value in the Customizer would produce `aria-label="  "` — technically non-empty but visually blank, causing the audit to fail. With `| strip`, whitespace-only collapses to `""` and the fallback chain fires correctly.

**index.json check:** Live section `banner_gallery_dKXLyg` has no `main_banner_aria_label` key saved → falls through to schema default `"Shop our latest lighting collection"` → renders correctly once deployed.

### 3. SEO Accessibility — Blog Post Links Descriptive Text

**Trigger:** Lighthouse SEO score 92/100 — "Links do not have descriptive text" flagged on blog post "READ MORE" links

**Files modified:**
- `snippets/article-card.liquid`
- `sections/blog-posts.liquid`

**Finding:** Lighthouse SEO checker requires descriptive text **inside** the anchor element itself — `aria-label` alone does not pass the check. All blog card CTAs contained only the text "READ MORE" with no context.

**Fix applied:** Added `<span class="visually-hidden"> about {{ article.title | escape }}</span>` inside each anchor. Visual appearance unchanged — sighted users still see "READ MORE". Screen readers and crawlers get full context.

| File | Change |
|------|--------|
| `article-card.liquid` | Added visually-hidden span + updated `aria-label` to use translation key with escaped title |
| `blog-posts.liquid` | Same pattern applied to placeholder blog post cards |

**Result:**

| Metric | Before | After |
|--------|--------|-------|
| Lighthouse SEO Score | 92 | 100 |

**Why `.visually-hidden` works:** Uses `clip: rect(0 0 0 0)` + `position: absolute` — text is in the DOM for crawlers/screen readers but invisible to sighted users. Unlike `display: none`, Lighthouse can read it.
