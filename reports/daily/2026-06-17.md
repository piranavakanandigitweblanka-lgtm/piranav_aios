### INP Investigation — Mobile Performance Root Cause Analysis
- **File:** `inp-investigation-2026-06-17.md`
- **Theme:** `theme_export__ledsone-co-uk-promotion-week-4-2-mega-digital__17JUN2026-1015am`
- **Scope:** Read-only — all JS files by page type, third-party scripts, app embeds, event listeners, long tasks
- **Critical findings:**
  1. `eval()` inside scroll handler (`theme.js:5,3422`) — JS re-parse on every scroll
  2. 4 unthrottled scroll handlers (`theme.js:627,982,3400,3414`) — no debounce/rAF
  3. Render-blocking inline `<script>` in `<head>` (`gsf-conversion-pixels.liquid`) — all major page types
  4. Body-level persistent listeners for popup — 4 types, no `once` flag (`theme.js:1478`)
  5. `fetch()` in 22 homepage click handlers blocking INP (`third-party-scripts.js:131-134`)
  6. TikTok Pixel + Clarity 5s unconditional fallback (`third-party-scripts.js:147`)
- **P1 fix:** Remove `eval()` from scroll path — biggest single INP gain
- **Risk:** Read-only investigation — no code changes made
