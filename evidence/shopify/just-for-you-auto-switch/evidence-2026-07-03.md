# Evidence: Just For You — Auto Product Switch
**Date:** 2026-07-03
**Section file:** `shopify_projects/electricalsone-theme/sections/just-for-you.liquid`

## Changes Made

### 1. Section tag — data attributes added (line 7–12)
Settings are passed from Liquid → HTML as data attributes so JS reads them at runtime.
```html
<section
  class="jfy-section"
  id="just-for-you-{{ section.id }}"
  data-auto-switch="{{ section.settings.enable_auto_switch }}"
  data-interval="{{ section.settings.switch_interval_seconds | times: 1000 }}"
  data-visible="{{ section.settings.visible_products_count }}">
```

### 2. Product card marker (line 28)
`data-jfy-card` added to each `.jfy-card` div so the rotation JS can select cards scoped to this section only.
```html
<div class="jfy-card" data-jfy-card>
```

### 3. CSS additions (after shimmer block)
```css
.jfy-card--hidden { display: none !important; }
.jfy-card--fade-in { animation: jfy-fadein 0.4s ease; }
@keyframes jfy-fadein { from { opacity:0; transform:translateY(6px); } to { opacity:1; transform:translateY(0); } }
```

### 4. JS rotation logic (inside existing IIFE, before closing `})()`)
- Reads `data-auto-switch`, `data-interval`, `data-visible` from section element
- If `enable_auto_switch` is false → exits immediately (all cards visible, no JS interference)
- If card count ≤ visible count → exits (all cards visible, no JS interference)
- Otherwise: hides cards outside group 0 on init, then `setInterval` rotates to next group every N ms
- Groups loop with modulo: after last group → back to group 0
- Each group reveal removes `jfy-card--hidden` and re-applies `jfy-card--fade-in` after a reflow
- Add to Cart button is inside each card — it is never touched by rotation logic

### 5. Schema settings added
| Setting ID | Type | Default | Range |
|---|---|---|---|
| `enable_auto_switch` | checkbox | true | — |
| `switch_interval_seconds` | range | 60 | 10–120 step 5 |
| `visible_products_count` | range | 6 | 2–12 step 1 |

## Fallback behaviour
If JavaScript is disabled or errors: `jfy-card--hidden` is never added, so all products remain visible in the normal grid. No server-side logic changed.
